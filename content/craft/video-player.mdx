## Quick Start

The `VideoPlayer` uses a compound component pattern. At minimum, you need a `Root` and a `Video`:

```tsx
import { VideoPlayer } from '@/app/craft/2025/video-player/component'

<VideoPlayer.Root>
  <VideoPlayer.Video src="/my-video.mp4" />
</VideoPlayer.Root>
```

Add controls by composing the parts you need:

```tsx
<VideoPlayer.Root className="relative">
  <VideoPlayer.Video src="/my-video.mp4" />
  <VideoPlayer.Controls className="absolute bottom-0">
    <VideoPlayer.PlayButton />
    <VideoPlayer.SeekSlider />
    <VideoPlayer.TimeDisplay />
  </VideoPlayer.Controls>
</VideoPlayer.Root>
```

## Styling

### Data attributes

Components expose data attributes for CSS styling. Use Tailwind's `data-[]` modifier:

```tsx
<VideoPlayer.PlayButton className="group">
  <PlayIcon className="hidden group-data-[paused]:block" />
  <PauseIcon className="hidden group-data-[playing]:block" />
  <ReplayIcon className="hidden group-data-[ended]:block" />
  <SpinnerIcon className="hidden group-data-[waiting]:block animate-spin" />
</VideoPlayer.PlayButton>
```

### CSS transitions

Components with enter/exit states (`Controls`, `Overlay`, `Poster`, `SeekSliderPreviewThumb`) expose `data-starting-style` and `data-ending-style` attributes for CSS transitions:

```tsx
<VideoPlayer.Controls
  className={cn(
    'opacity-100 translate-y-0',
    'transition-[opacity,transform] duration-300',
    'data-[starting-style]:opacity-0 data-[starting-style]:translate-y-2',
    'data-[ending-style]:opacity-0 data-[ending-style]:translate-y-2',
  )}
/>
```

Or with plain CSS:

```css
.controls {
  opacity: 1;
  transform: translateY(0);
  transition: opacity 300ms, transform 300ms;
}

.controls[data-starting-style],
.controls[data-ending-style] {
  opacity: 0;
  transform: translateY(8px);
}
```

### CSS animations

You can use CSS keyframe animations with `data-open` and `data-closed` attributes.

With Tailwind CSS v4 + [`tw-animate-css`](https://github.com/Wombosvideo/tw-animate-css/tree/main):

```tsx
<VideoPlayer.Controls
  className={cn(
    'data-[open]:animate-in data-[closed]:animate-out',
    'data-[open]:fade-in data-[closed]:fade-out',
    'data-[open]:slide-in-from-bottom-4 data-[closed]:slide-out-to-bottom-4',
    'duration-300',
  )}
/>
```

Or with plain CSS:

```css
.controls[data-open] {
  animation: fade-in 300ms ease-out, slide-in 300ms ease-out;
}

.controls[data-closed] {
  animation: fade-out 300ms ease-out, slide-out 300ms ease-out;
}

@keyframes fade-in {
  from { opacity: 0; }
  to { opacity: 1; }
}

@keyframes fade-out {
  from { opacity: 1; }
  to { opacity: 0; }
}

@keyframes slide-in {
  from { transform: translateY(1rem); }
  to { transform: translateY(0); }
}

@keyframes slide-out {
  from { transform: translateY(0); }
  to { transform: translateY(1rem); }
}
```

### JavaScript animations

For JavaScript animation libraries like [Motion](https://motion.dev), use `keepMounted` and the `render` prop to compose with `motion` components:

```tsx
import { motion, AnimatePresence } from 'motion/react'

function Player() {
  const [open, setOpen] = React.useState(true)

  return (
    <VideoPlayer.Root>
      <VideoPlayer.Video src="/video.mp4" />
      <AnimatePresence>
        {open && (
          <VideoPlayer.Controls
            keepMounted
            render={(props) => (
              <motion.div
                {...props}
                initial={{ opacity: 0, y: 10 }}
                animate={{ opacity: 1, y: 0 }}
                exit={{ opacity: 0, y: 10 }}
              />
            )}
          >
            {/* controls */}
          </VideoPlayer.Controls>
        )}
      </AnimatePresence>
    </VideoPlayer.Root>
  )
}
```

The `keepMounted` prop ensures the element stays in the DOM during exit animations, and the `render` prop lets you compose with Motion's `motion` components for full animation control.

## Composition

All components support a `render` prop for full control:

```tsx
<VideoPlayer.PlayButton
  render={(props, state) => (
    <button {...props} className="my-button">
      {state.playing ? 'Pause' : state.ended ? 'Replay' : 'Play'}
    </button>
  )}
/>
```

|-----|--------|

| `M` | Toggle mute |
| `F` | Toggle fullscreen |
| `ArrowLeft` | Seek -5s |
| `ArrowRight` | Seek +5s |
| `ArrowUp` | Volume +10% |
| `ArrowDown` | Volume -10% |
| `0-9` | Seek to 0%-90% |

## API reference

### Root

Groups all parts of the video player and manages shared state including playback, volume, idle detection, and keyboard shortcuts. Renders a `<div>` element.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `keyboardShortcuts` | `'focused' \| 'global' \| 'none'` | `'focused'` | Keyboard shortcuts scope. |
| `idleTimeout` | `number` | `3000` | Time in ms before player is considered idle. |
| `preventIdleWhenPaused` | `boolean` | `false` | Prevents idle state while paused. |
| `hideCursorWhenIdle` | `'always' \| 'fullscreen' \| false` | `'fullscreen'` | Cursor auto-hiding behavior. |
| `defaultPlaying` | `boolean` | `false` | Initial playing state. |
| `defaultVolume` | `number` | `1` | Initial volume (0-1). |
| `defaultMuted` | `boolean` | `false` | Initial muted state. |
| `defaultPlaybackRate` | `number` | `1` | Initial playback rate. |
| `playing` | `boolean` | - | Controlled playing state. |
| `onPlayingChange` | `(playing: boolean) => void` | - | Playing state change callback. |
| `volume` | `number` | - | Controlled volume. |
| `onVolumeChange` | `(volume: number) => void` | - | Volume change callback. |
| `muted` | `boolean` | - | Controlled muted state. |
| `onMutedChange` | `(muted: boolean) => void` | - | Muted state change callback. |
| `onEnded` | `() => void` | - | Video ended callback. |
| `onFullscreenChange` | `(fullscreen: boolean) => void` | - | Fullscreen change callback. |

| Attribute | Description |
|-----------|-------------|
| `data-playing` | Video is playing. |
| `data-paused` | Video is paused. |
| `data-ended` | Video has ended. |
| `data-waiting` | Video is buffering. |
| `data-seeking` | Video is seeking. |
| `data-fullscreen` | In fullscreen mode. |
| `data-pip` | In picture-in-picture mode. |
| `data-muted` | Audio is muted. |

### Track

Defines a caption or subtitle track. Must be placed inside `Video`. Renders a `<track>` element.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `kind` | `TextTrackKind` | `'captions'` | Track kind. |
| `src` | `string` | - | Source URL for VTT file. |
| `label` | `string` | - | Human-readable label. |
| `srcLang` | `string` | - | Language code (e.g., `'en'`). |
| `default` | `boolean` | `false` | Set as default track. |
